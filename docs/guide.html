kvfs м•„нӮӨн…ҚмІҳ к°Җмқҙл“ң

л¶„мӮ° object storage к°Җ мһ‘лҸҷн•ҳлҠ” мӣҗлҰ¬лҘј л”°лқјк°ҖлҠ” к°Җмқҙл“ң. м§§мқҖ reference к°Җ н•„мҡ”н•ҳл©ҙ ARCHITECTURE.md вҖ” мқҙ л¬ём„ңлҠ” мқҪмңјл©° л”°лқјк°ҖлҠ” walkthrough.

0лҲ„кө¬лҘј мң„н•ң л¬ём„ңмқёк°Җ

  • л¶„мӮ° storage к°Җ м–ҙл–»кІҢ л§Ңл“Өм–ҙм§ҖлҠ”м§Җ мҪ”л“ңлҘј мқҪмңјл©° мқҙн•ҙн•ҳкі  мӢ¶мқҖ к°ңл°ңмһҗ
  • Ceph В· MinIO В· S3 мқҳ лҸҷмһ‘мқ„ мһ‘мқҖ лӘЁлҚёлЎң мһ¬нҳ„н•ҙліҙкі  мӢ¶мқҖ н•ҷмғқ
  • мҳӨл„ҲмӢӯВ·мқјкҙҖм„ұВ·мҡҙмҳҒм„ұмқҳ нҠёл Ҳмқҙл“ңмҳӨн”„лҘј мІҙк°җн•ҳкі  мӢ¶мқҖ SRE

м „м ң м§ҖмӢқ: HTTP, hash н•ЁмҲҳ, goroutine, bbolt/SQLite к°ҷмқҖ KV м ҖмһҘмҶҢк°Җ л¬ҙм—Үмқём§Җ.
н•„мҡ” м—ҶлҠ” кІғ: Ceph CRUSH, Raft л…јл¬ё м •лҸ…, Reed-Solomon мҲҳн•ҷ.

  • к°қмІҙ лӘЁлҚё: replication (default 3-way) лҳҗлҠ” Reed-Solomon EC вҖ” X-KVFS-EC: K+M н—ӨлҚ”лЎң PUT л§ҲлӢӨ м„ нғқ.
  • л©”нғҖлҚ°мқҙн„°: bbolt лӢЁмқј нҢҢмқј. objects лІ„нӮ·мқҙ single source of truth.
  • HA: л©ҖнӢ° edge вҖ” primary 1 + read-replica N (snapshot pull) лҳҗлҠ” Raft-style auto-elected leader.
  • мҡҙмҳҒм„ұ: edge к°Җ мһҗкё° cluster мқҳ м •л ¬В·мІӯмҶҢВ·ліөкө¬В·snapshotВ·heartbeat лҘј мҠӨмҠӨлЎң н•ңлӢӨ.

2кё°ліё кө¬мЎ°

kvfs-edge вҖ” кІҢмқҙнҠёмӣЁмқҙ

  • нҸ¬нҠё: :8000 (HTTP лҳҗлҠ” HTTPS, ADR-029).
  • м—ӯн• : нҒҙлқјмқҙм–ёнҠё мҡ”мІӯ л°ӣмқҢ вҶ’ URL м„ңлӘ… кІҖмҰқ вҶ’ chunker/EC мІҳлҰ¬ вҶ’ placement вҶ’ DN fanout вҶ’ л©”нғҖ м ҖмһҘ.
  • мғҒнғң нҢҢмқј: <EDGE_DATA_DIR>/edge.db (bbolt) + м„ нғқм Ғ <EDGE_SNAPSHOT_DIR>/ + м„ нғқм Ғ WAL нҢҢмқј.
  • мҷёл¶Җ мқҳмЎҙ: DN л“Ө. к·ё мҷё 0 (no etcd, no zookeeper, no DB).

kvfs-dn вҖ” мҠӨнҶ лҰ¬м§Җ л…ёл“ң

  • нҸ¬нҠё: :8080 (HTTP лҳҗлҠ” HTTPS).
  • м—ӯн• : chunk лӢЁмң„ PUT/GET/DELETE. н•ң chunk = н•ң нҢҢмқј.
  • мғҒнғң нҢҢмқј кІҪлЎң: <DN_DATA_DIR>/chunks/<sha256[0:2]>/<rest-of-sha>. мІ« 2мһҗлҰ¬ prefix лЎң л””л үнҶ лҰ¬ л¶„мӮ°.
  • мҷёл¶Җ мқҳмЎҙ: 0. DN лҒјлҰ¬ м„ңлЎң лӘЁлҰ„.

лҚ°мқҙн„° л№„лҢҖм№ӯ

edge к°Җ лҳ‘лҳ‘н•ҳкі , dn мқҖ л©ҚмІӯн•ҳлӢӨ. dn мқҖ мһҗкё°к°Җ м–ҙлҠҗ object мқҳ мқјл¶Җмқём§Җ лӘЁлҘҙкі , м–ҙлҠҗ DN л“Өкіј н•ң stripe мқём§ҖлҸ„ лӘЁлҘёлӢӨ. лӘЁл“  мқҳмӮ¬кІ°м •мқҖ edge к°Җ н•ңлӢӨ.

мқҙ л№„лҢҖм№ӯмқҙ kvfs лӢЁмҲңн•Ёмқҳ н•өмӢ¬мқҙлӢӨ. dn мқҙ "self-organizing" мқҙ м•„лӢҲлқј, edge к°Җ placement м•Ңкі лҰ¬мҰҳмңјлЎң кІ°м • н•ҳкі  dn мқҖ byte container мқј лҝҗ.

3н•өмӢ¬ м»Ём…ү 5к°ң

3.1 Content-addressable chunk ADR-005

chunk_id = hex(sha256(chunk_bytes))
  • chunk мқҳ лӮҙмҡ© мқҙ кі§ мЈјмҶҢ. к°ҷмқҖ byte лҠ” м–ҙл””м„ң лӮҳмҷҖлҸ„ к°ҷмқҖ chunk_id.
  • л¬ҙлЈҢ dedup: л‘җ к°қмІҙк°Җ к°ҷмқҖ chunk лҘј кіөмң н•ҳл©ҙ disk мң„м—җ н•ҳлӮҳл§Ң мЎҙмһ¬.
  • л¬ҙлЈҢ integrity: GET мӢң sha256 лӢӨмӢң кі„мӮ°н•ҙм„ң ID мҷҖ л№„көҗ (chunker.Join).
  • DN мқҖ ID вҶ’ byte лҘј м ҖмһҘл§Ң н•ңлӢӨ. ID к°Җ м–ҙл–»кІҢ мғқкІјлҠ”м§Җ лӘЁлҘёлӢӨ.

3.2 Rendezvous Hashing (HRW) ADR-009

к°ҷмқҖ chunk лҠ” н•ӯмғҒ к°ҷмқҖ DN 3к°ң м—җ к°Җм•ј н•ңлӢӨ (мһ¬нҳ„ к°ҖлҠҘ + л©”нғҖ м Ҳм•Ҫ). DN мқҙ м¶”к°Җ/м ңкұ°лҗҳм–ҙлҸ„ лҢҖл¶Җл¶„мқҳ chunk лҠ” к·ё мһҗлҰ¬ (consistent).

м•Ңкі лҰ¬мҰҳ:

for each DN:
  score = hash(chunk_id || "|" || dn_id)
choose top R by score
  • N к°ң DN мӨ‘ м–ҙлҠҗ R к°ңлЎң к°Ҳм§Җ вҖ” м–ҙл””м„ңлӮҳ лҸҷмқјн•ң кІ°м •.
  • DN N вҶ’ N+1 мӢң нҸүк·  R/(N+1) chunk л§Ң мқҙлҸҷ.
  • Cassandra/DynamoDB мқҳ ring кё°л°ҳ мқјкҙҖм„ұ н•ҙмӢұкіј к°ҷмқҖ лӘ©н‘ң, лӢӨлҘё м•Ңкі лҰ¬мҰҳ. мҪ”л“ң ~30 мӨ„.

3.3 Reed-Solomon Erasure Coding ADR-008

K data shards + M parity shards = K+M к°ң shards к°Җ stripe н•ҳлӮҳ
  • л””мҠӨнҒ¬ мӮ¬мҡ©лҹү: replication 3Г— вҶ’ EC (10+4) 1.4Г—. к°ҷмқҖ лӮҙкө¬лҸ„, 2л°° нҡЁмңЁ.
  • м–ҙл–Ө K к°ң shards л§Ң мӮҙм•„мһҲм–ҙлҸ„ мӣҗліё ліөкө¬ к°ҖлҠҘ (Reconstruct).
  • GF(2^8) мң„м—җм„ң Vandermonde н–үл ¬ + Gauss-Jordan мңјлЎң from-scratch кө¬нҳ„.
  • per-PUT мҳөм…ҳ: X-KVFS-EC: 4+2 н—ӨлҚ” мһҲмңјл©ҙ EC, м—Ҷмңјл©ҙ replication.

3.4 Content-defined Chunking (FastCDC) ADR-018

кі м • 4 MiB chunking (ADR-011) мңјлЎң мһҗлҘҙл©ҙ 1 byte л§Ң лҒјмӣҢл„Јм–ҙлҸ„ лӘЁл“  chunk_id к°Җ лӢ¬лқјм ё dedup 0%. CDC лҠ” chunk кІҪкі„лҘј content лЎң кІ°м •н•ҙм„ң shift м—җ к°•н•ҳлӢӨ.

fp = 0
for each byte b in stream:
  fp = (fp << 1) + GearTable[b]
  if fp & mask == 0:  cut here.
  • rolling hash вҶ’ к°ҷмқҖ 32-byte window к°Җ л“Өм–ҙмҳӨл©ҙ к°ҷмқҖ fp вҶ’ к°ҷмқҖ cut.
  • 99% лҸҷмқј + 1% лӢ¬лқјм§„ л‘җ к°қмІҙк°Җ ~99% chunk лҘј кіөмң  (Borg/Restic н•өмӢ¬ кё°лІ•).
  • мҳөм…ҳ: EDGE_CHUNK_MODE=cdc. кё°ліёмқҖ fixed (Season 2 нҳёнҷҳ).

3.5 HMAC Presigned URL ADR-007

signature = hex(HMAC-SHA256(secret, "GET\nbucket/key\nexpiry"))
URL: /v1/o/bucket/key?sig=<hex>&exp=<unix-ts>
  • м„ңлӘ… 1лІҲ л°ңкёү вҶ’ л§ҢлЈҢк№Ңм§Җ Client к°Җ м§Ғм ‘ GET, edge лЎң л§ӨлІҲ мқёмҰқ м•Ҳ мҳҙ.
  • S3 presigned URL кіј к°ҷмқҖ нҢЁн„ҙ. ~90 мӨ„ мҪ”л“ң.
  • kid нҡҢм „ (ADR-028) мңјлЎң нӮӨ л…ём¶ң мӢң л¬ҙмӨ‘лӢЁ көҗмІҙ.

4к°қмІҙ лқјмқҙн”„мӮ¬мқҙнҒҙ

4.1 PUT (replication, default)

sequenceDiagram participant C as Client participant E as edge participant D as DN Г—R participant B as bbolt C->>E: PUT /v1/o/b/k?sig=... E->>E: verify HMAC E->>E: chunker.Split(body) вҶ’ chunks loop к°Ғ chunk E->>E: id = sha256(chunk) E->>E: targets = HRW.Pick(id, R) E->>D: parallel PUT (Г—R) D-->>E: ack Note over E,D: R/2+1 ack мӢң success end E->>B: objects[b/k] = ObjectMeta E-->>C: 201 Created

н•өмӢ¬:

  • Quorum: R=3 мқј л•Ң 2 ack мӢң м„ұкіө. 3лІҲм§ёлҠ” background.
  • Streaming (ADR-017): body лҘј л©”лӘЁлҰ¬м—җ лӢӨ м•Ҳ л°ӣкі  chunker.Reader лЎң н•ң chunk м”© мқҪкі  ліҙлӮҙкі  лІ„лҰ°лӢӨ вҶ’ edge RAM = chunkSize мҷҖ л¬ҙкҙҖ.

4.2 PUT (EC) ADR-008

sequenceDiagram participant C as Client participant E as edge participant D as DN Г—(K+M) C->>E: PUT ... + X-KVFS-EC: K+M loop к°Ғ stripe E->>E: split вҶ’ K data shards E->>E: RS Encode вҶ’ +M parity E->>E: HRW.PlaceN(stripe_id, K+M) E->>D: parallel PUT K+M shards D-->>E: ack end E->>E: bbolt: ObjectMeta{EC, Stripes} E-->>C: 201 Created

4.3 GET (replication / EC кіөнҶө нҳ•нғң)

edge: bbolt[b/k] вҶ’ ObjectMeta
  в”ңв”Җ replication: for each chunk вҶ’ replicas[0] мӢңлҸ„ вҶ’ мӢӨнҢЁмӢң [1] вҶ’ ...
  в”Ӯ              м„ұкіө byte вҶ’ sha256 мһ¬кі„мӮ° вҶ’ ID л§Өм№ҳ нҷ•мқё вҶ’ write
  в”Ӯ
  в””в”Җ EC: for each stripe вҶ’
         alive shards м—җм„ң K к°ң fetch (replicas[0] мҡ°м„ )
         вҶ’ RS Reconstruct вҶ’ data shards K к°ң ліөмӣҗ
         вҶ’ chunker.Join вҶ’ write

GET лҠ” мһҗк°Җ м№ҳмң  м•Ҳ н•Ё вҖ” мҶҗмғҒ chunk л°ңкІ¬н•ҳл©ҙ к·ёлғҘ лӢӨмқҢ replica лЎң fallback. мҳҒмҶҚ ліөкө¬лҠ” лі„лҸ„ worker (ADR-025 EC repair).

4.4 DELETE

edge: bbolt[b/k] вҶ’ ObjectMeta вҶ’ replicas/shards DN л“Өм—җ DELETE лі‘л ¬
      bbolt м—җм„ң (b/k) м ңкұ°

мқҙлҜё м•Ҳ мӮ¬лҠ” chunk к°Җ л””мҠӨнҒ¬м—җ лӮЁмңјл©ҙ вҮ’ surplus chunk вҶ’ GC к°Җ мІӯмҶҢ (ADR-012).

5м•Ңкі лҰ¬мҰҳ к№Ҡмқҙ

5.1 Quorum м •нҷ•нһҲ

  • replicas R = 3 (default) вҶ’ write quorum = вҢҠR/2вҢӢ+1 = 2.
  • 2 DN м—җ PUT м„ұкіө вҶ’ edge к°Җ 200 мқ‘лӢө нӣ„ 3лІҲм§ё DN мқҖ best-effort.
  • 3лІҲм§ёк°Җ мӢӨнҢЁн•ҙлҸ„ лҚ°мқҙн„°лҠ” 2к°ңм—җ м•Ҳм „. rebalance worker (ADR-010) мқҙ лӮҳмӨ‘м—җ R нҡҢліө.
  • read quorum = 1 вҖ” н•ң DN л§Ң мӮҙм•„мһҲм–ҙлҸ„ GET м„ұкіө.

5.2 Reed-Solomon м§ҒкҙҖ

"K shards лЎң K+M shards лҘј л§Ңл“Өкі , м–ҙл–Ө K shards л§Ң мһҲм–ҙлҸ„ мӣҗліё ліөмӣҗ."

  • generator matrix G = [I_K | V] вҖ” мң„ KГ—K лҠ” лӢЁмң„н–үл ¬ (=data к·ёлҢҖлЎң), м•„лһҳ MГ—K лҠ” Vandermonde (=parity кі„мӮ°).
  • shard мҶҗмӢӨ N к°ң вҶ’ G м—җм„ң к·ё row л“Ө л№јкі  KГ—K м •л°©н–үл ¬ G' вҶ’ Gauss-Jordan мңјлЎң G'вҒ»В№ кө¬н•Ё вҶ’ мӣҗліё ліөмӣҗ.
  • GF(2^8) = 1 byte лӢЁмң„ мӮ°мҲ . кіұм…ҲмқҖ log/exp н‘ң lookup, лҚ§м…ҲмқҖ XOR.

5.4 HRW мқҳ "consistent" мҰқлӘ… м§ҒкҙҖ

DN N вҶ’ N+1 м¶”к°Җ мӢң:

  • мғҲ DN мқҳ м җмҲҳк°Җ кё°мЎҙ top-R м•Ҳм—җ л“Өм–ҙк°ҖлҠ” chunk л§Ң мҳҒн–Ҙ л°ӣмқҢ.
  • chunk лі„лЎң м җмҲҳ л¶„нҸ¬лҠ” к· л“ұ вҶ’ мғҲ DN мқҙ top-R мқј нҷ•лҘ  = R/(N+1).
  • мҳҒн–Ҙ л°ӣмқҖ chunk лҠ” 1к°ң мқҙлҸҷ (к°ҖмһҘ лӮ®м•ҳлҚҳ кё°мЎҙ DN 1к°ң л°Җл ӨлӮЁ).
  • кё°лҢ“к°’ мқҙлҸҷлҹү = м „мІҙ chunk Г— R/(N+1) вҖ” к· нҳ•м җ.

6мҡҙмҳҒм„ұ (Season 3)

edge к°Җ мҠӨмҠӨлЎң мһҗкё° cluster лҘј лҸҢліёлӢӨ. мҷёл¶Җ cron, мҷёл¶Җ controller м—ҶмқҢ.

6.1 Auto-trigger ADR-013

StartAuto(ctx):
  go ticker(EDGE_AUTO_REBALANCE_INTERVAL) вҶ’ executeRebalance()
  go ticker(EDGE_AUTO_GC_INTERVAL)        вҶ’ executeGC()

л‘җ ticker к°Җ к°ҷмқҖ mutex кіөмң  вҶ’ лҸҷмӢң мӢӨн–ү кёҲм§Җ. мҲҳлҸҷ admin call лҸ„ к°ҷмқҖ mutex мңјлЎң м§Ғл ¬нҷ”. Ring buffer (size 32) м—җ кІ°кіј м ҖмһҘ вҶ’ /v1/admin/auto/status лЎң мЎ°нҡҢ.

6.2 Rebalance worker ADR-010 В· ADR-024

  • Replicated chunk: ChunkRef.replicas к°Җ placement.Pick кІ°кіјмҷҖ лӢӨлҘҙл©ҙ вҶ’ лҲ„лқҪ DN мңјлЎң copy вҶ’ л©”нғҖ м—…лҚ°мқҙнҠё вҶ’ мһүм—¬ DN мқҳ chunk лҠ” GC лҢҖмғҒ.
  • EC stripe (ADR-024): set-based л№„көҗ. нҳ„мһ¬ shard мң„м№ҳ set vs мқҙмғҒм Ғ set. м°Ём§‘н•©мңјлЎң мөңмҶҢ мқҙлҸҷл§Ң.

6.3 GC ADR-012

surplus chunk = л©”нғҖм—җм„ң м°ёмЎ° м•Ҳ лҗҳлҠ”лҚ° DN л””мҠӨнҒ¬м—җ лӮЁмқҖ chunk.
